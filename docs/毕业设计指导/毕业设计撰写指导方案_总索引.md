## 使用建议

### 🎯 快速开始（首次阅读）

1. **先读第1部分** - 了解整体论文结构和前三章框架
2. **重点读第4部分第8章** - 学习实验评估的标准写法（最重要）
3. **参考第5部分附录C** - 掌握学术写作话术技巧
4. **浏览其他章节** - 根据自己项目特点选择性深入

### 📝 撰写阶段使用方式

**阶段1：搭建框架（1-2天）**
- 参考第1部分的目录结构，在Word/LaTeX中创建章节框架
- 完成摘要、绪论的初稿（参考第1部分的行文示例）

**阶段2：填充设计章节（3-5天）**
- 按照第2、3部分的指导，逐章撰写系统设计内容
- 注意：每节包含【行文示例】、代码片段、图表建议，直接参考

**阶段3：前端与实验（3-5天）**
- 参考第4部分撰写前端设计（如有）
- **重点**：按照第4部分第8章的实验设计模板完成实验评估
  - 实验环境配置表
  - 对照组设置
  - 数据表格+误差线
  - 结果分析+原因解释

**阶段4：总结与润色（1-2天）**
- 参考第5部分第9章撰写总结与展望
- 使用附录C的检查清单全文自查
- 检查图表编号、参考文献格式

### 🎤 答辩准备阶段

- 阅读第5部分附录D《答辩准备建议》
- 按照推荐的PPT结构制作幻灯片
- 演练常见答辩问题（Q&A预演）

## 核心撰写原则

### ✅ 必须做到

1. **数据支撑每个结论** - 避免"性能很好"等模糊表述，用具体数据说话
2. **图表清晰规范** - 编号连续、标题位置正确、分辨率足够
3. **实验有对照组** - 通过对比体现优势
4. **术语首次出现有定义** - 中英文全称 + 简要解释
5. **引用充分** - 不少于20篇参考文献，近5年占比>50%

### ❌ 避免错误

1. **避免主观臆断** - ❌"本系统非常优秀" ✅"实验表明本系统延迟降低12%"
2. **避免口语化** - ❌"跑得飞快" ✅"性能开销低"
3. **避免只讲功能** - 要讲设计思路、算法原理、性能优化
4. **避免大段代码** - 关键代码片段即可，完整代码放附录或GitHub
5. **避免图表模糊** - 矢量图优于位图，字号不小于10pt

## 常见问题FAQ

**Q1: 我的项目没有前端，怎么办？**  
A: 可以省略第7章Qt前端设计，将第8章实验评估扩充至10-12页。重点展示后端算法与性能优化。

**Q2: 实验数据不够理想，怎么办？**  
A: (1) 如实呈现数据，在分析中解释原因；(2) 通过对比基线（Baseline）突出相对优势；(3) 在局限性中坦诚说明。切忌造假数据。

**Q3: 参考文献不够20篇怎么办？**  
A: (1) 补充CXL标准文档、Intel技术手册；(2) 引用经典教材（如CAQA）；(3) 搜索近3年顶会论文（ASPLOS/ISCA/OSDI）。

**Q4: 怎么证明系统的创新性？**  
A: (1) 与gem5/DRAMSim对比易用性（配置时间）；(2) 与手工计算对比精度；(3) 用真实案例（YOLOv8）体现实用价值。

**Q5: 答辩时被问到不会的问题怎么办？**  
A: 坦诚说"这是一个很好的问题，目前我还没有深入研究，未来可以作为改进方向"。切忌不懂装懂。
