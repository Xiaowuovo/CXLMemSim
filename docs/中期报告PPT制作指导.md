## 可能的提问及应对

### Q1: 你的系统与gem5有什么区别？

**回答**:
```
gem5是通用的处理器架构仿真器，覆盖CPU/Cache/Memory全栈，配置复杂。
CXLMemSim专注于CXL内存子系统，特点是：
1. 提供可视化拓扑编辑器，5分钟完成配置（gem5需要30分钟写Python脚本）
2. 支持虚拟机环境，无需特殊硬件（gem5的PEBS模式需要物理机）
3. 内置A/B对比功能，形成闭环工作流

定位不同：gem5适合研究CPU微架构，CXLMemSim适合研究内存拓扑配置。
```

### Q2: 延迟建模的精度如何保证？

**回答**:
```
我们通过以下方式验证精度：
1. 对比文献[DirectCXL, USENIX ATC'22]报道的实测数据：
   文献230ns，我们仿真228ns，误差<1%
2. 延迟公式基于CXL官方规范和PCIe 5.0标准
3. M/M/1排队模型是业界标准的拥塞建模方法

未来如果获得真实CXL硬件，可以补充硬件在环验证。
```

### Q3: 目前完成度如何？能按时完成吗？

**回答**:
```
核心功能已100%完成，包括：
- 后端仿真引擎（Tracer/Topology/Latency/Analyzer）
- 前端可视化（编辑器/监控/对比）
- 基础实验（拓扑对比/映射策略/YOLOv8案例）

剩余工作主要是：
- 补充2-3组实验（可扩展性测试）
- 论文撰写（已有设计文档可参考，预计2周完成）

时间充足，有信心按时完成。
```

### Q4: YOLOv8案例的实用价值在哪？

**回答**:
```
YOLOv8案例证明了系统对真实AI负载的建模能力：
1. 通过perf mem录制真实Trace（120万次访问）
2. 回放Trace评估不同CXL配置对推理性能的影响
3. 发现DRAM + 小容量CXL混合方案可提升帧率56.8%

实用价值：为边缘AI服务器的内存配置提供量化决策依据，
证明CXL扩展内存是一种低成本高性能方案（相比扩容DRAM）。
```

1. **PowerPoint**（推荐）
   - 模板资源丰富
   - 动画效果易制作
   - 兼容性最好

2. **Keynote**（Mac用户）
   - 视觉效果精美
   - 过渡动画流畅

3. **Beamer LaTeX**（学术风格）
   - 适合数学公式多的内容
   - 排版专业

### 配色方案

**主色调**（科技风格）:
- 主色：#4FC3F7（浅蓝）
- 辅色：#FF9800（橙色，用于强调）
- 背景：深色主题（#1A1A1A）或浅色主题（#F5F5F5）
- 文字：深色背景用白色/浅灰，浅色背景用深灰

**避免**:
- 过于鲜艳的配色（刺眼）
- 纯红色背景（警告色）
- 低对比度组合（如灰底灰字）

### 字体选择

**中文**:
- 标题：微软雅黑 Bold / 思源黑体
- 正文：微软雅黑 / 苹方

**英文**:
- 标题：Arial Black / Roboto Bold
- 正文：Arial / Segoe UI
- 代码：Consolas / Fira Code

**字号**:
- 标题：32-40pt
- 正文：20-24pt（确保最后一排能看清）
- 注释：14-16pt

## Demo演示准备

### 录制演示视频（推荐）

**优点**:
- 避免现场操作失误
- 可以加速播放节省时间
- 可添加字幕标注

**录制工具**:
- Windows: OBS Studio（免费）
- Mac: QuickTime Player / ScreenFlow
- 跨平台: Camtasia（付费但功能强大）

**录制内容**:
1. 拓扑编辑全流程（拖拽-连线-配置-自动布局）30秒
2. 仿真启动与实时监控 20秒
3. 基准固定与对比 20秒

**视频规格**:
- 分辨率：1920x1080
- 帧率：30fps
- 格式：MP4（兼容性最好）
- 时长：每段控制在20-30秒

### 现场演示准备

如果选择现场操作：
- 提前在演示机器上测试（确保Qt运行正常）
- 准备好预配置的拓扑文件（避免现场从零开始）
- 设置较大的Epoch时长，避免数据刷新太快
- 关闭不必要的后台程序（避免卡顿）

## 总结

中期报告PPT的核心目标是**证明项目进展顺利**，重点在于：

1. **功能演示为主** - 用截图/视频展示实际运行效果
2. **数据支撑进度** - 用完成度百分比、性能指标证明工作量
3. **坦诚汇报问题** - 已知问题 + 解决方案体现严谨
4. **明确后续计划** - 时间线清晰，让导师放心

**关键建议**:
- 控制时间在10-12分钟（留2分钟Q&A）
- 提前演练2-3次
- 准备Demo视频作为保险
- 自信从容，问题不会的如实说明
