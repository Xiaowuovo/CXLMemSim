## 📚 科研文献调研总结

### 1. CXL性能研究的学术现状

#### 重要学术论文

1. **"TPP: Transparent Page Placement for CXL-Enabled Tiered Memory"** (ASPLOS 2023)
   - 研究内容：CXL内存的页面放置策略
   - 关键发现：智能页面迁移可降低20-30%的平均延迟
   - 对本项目的启示：需要支持不同内存分配策略的模拟

2. **"Pond: CXL-Based Memory Pooling Systems for Cloud Platforms"** (ASPLOS 2023)
   - 研究内容：数据中心级别的CXL内存池化
   - 关键发现：池化架构可提高内存利用率40%
   - 对本项目的启示：需要支持多设备拓扑和资源共享模拟

3. **"Memory Expansion and Data Placement in CXL-Based Systems"** (ISCA 2022)
   - 研究内容：CXL内存扩展的性能影响
   - 关键发现：延迟和带宽的trade-off曲线
   - 对本项目的启示：需要提供参数扫描和敏感性分析工具

#### 业界白皮书

1. **Intel CXL Memory Expansion White Paper**
   - CXL Gen5: 170ns延迟，64GB/s带宽
   - CXL Gen4: 250ns延迟，32GB/s带宽
   - 推荐场景：AI训练、内存数据库、图计算

2. **AMD CXL Technology Overview**
   - 强调拥塞控制的重要性
   - 提供了M/M/1排队模型的验证数据

#### 需求1：参数空间探索

**场景**：

**改进方向**：

#### 需求2：可重复性保证

**场景**：
- 论文审稿人要求提供可重复的实验环境
- 需要记录所有实验参数、随机种子、系统版本

**当前支持**：
- ✅ JSON配置文件
- ✅ 实验结果导出
- ❌ 完整的可重复性元数据

**改进方向**：

#### 需求3：对比分析

**场景**：

**当前支持**：
- ✅ 单次模拟结果显示
- ❌ 多组实验对比视图

**改进方向**：
- 添加"对比模式"，可同时加载多个结果
- 提供对比图表（柱状图、折线图）

#### 需求4：实时调参

**场景**：
- 在模拟运行时动态调整参数（如延迟、带宽）

**当前支持**：
- ❌ 参数需要停止模拟后修改

**改进方向**：

### 增强2：百分位延迟统计

**实现方案**：

```cpp
// 在EpochStats中添加
struct EpochStats {
    // 现有字段...
    double avg_latency_ns;
    double max_latency_ns;
    double min_latency_ns;
    
    // 新增字段
    double p50_latency_ns;  // 中位数
    double p90_latency_ns;  // 90百分位
    double p99_latency_ns;  // 99百分位
    double p999_latency_ns; // 99.9百分位
    
    // 延迟分布（用于计算百分位）
    std::vector<double> latency_samples;
};
```

**科研价值**：
- 百分位延迟是评估尾延迟的关键指标
- Google、Facebook等公司的SLA都基于P99延迟
- 学术论文中必需的统计量

### 建议2：使用SQLite存储实验历史

```cpp
// 数据库表结构
CREATE TABLE experiments (
    id INTEGER PRIMARY KEY,
    name TEXT,
    timestamp TEXT,
    config_json TEXT,
    results_json TEXT,
    metadata_json TEXT
);

// 查询历史实验
SELECT name, timestamp, avg_latency 
FROM experiments 
ORDER BY timestamp DESC 
LIMIT 10;
```
